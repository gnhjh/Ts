OBJECTIVE
Strengthen JavaScript / TypeScript fundamentals by implementing
form handling, validation logic, and clean event-driven flow.

CORE FOCUS

- Form handling
- Validation
- Event management
- Clear logical flow

---

REQUIREMENTS

User Model

- fullName: string
- email: string
- password: string

Validation Rules

- fullName → minimum 3 characters
- email → must include "@"
- password → minimum 6 characters

Behavior

- Store users in an in-memory array
- Allow adding a new user
- Allow displaying all users

Output

- If validation fails → display which field is invalid
- If validation passes → user is added successfully

---

CONSTRAINTS

- JavaScript / TypeScript only
- No external libraries
- No advanced UI required (console-based implementation is acceptable)

---

EVALUATION CRITERIA

- Code readability
- Logical flow
- Simplicity over complexity
- Proper TypeScript usage
- Clear separation between validation and business logic

---

GOAL
Build a minimal but structured solution.
Treat this as production-quality code — not a quick script.
