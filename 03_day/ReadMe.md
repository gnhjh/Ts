# 🎯 משימה – User Manager

בתרגיל זה אתה מרחיב את מערכת ההרשמה שלך עם מחיקה, בדיקת כפילויות ו-LocalStorage.

---

## 📋 דרישות חדשות

### 1️⃣ בדיקת משתמש קיים
- **אסור להוסיף משתמש עם אותו email אם הוא קיים בערכת**
- עם הוספת משתמש → בדיקה אוטומטית
- אם כבר קיים → תוצג שגיאה מתאימה ❌

> 👉 **כאן אתה כבר מתחיל לחשוב על data integrity**

---

### 2️⃣ מחיקת משתמש
לכל שורה בטבלה יהיה כפתור **Delete**:

- **בלחיצה:**
  1. המשתמש יימחק מה-array
  2. השורה תימחק מה-DOM
  3. ה-localStorage יתעדכן
  4. הטבלה תרינדר מחדש

> 👉 **כאן אתה מתחיל לעבוד עם Event Delegation**

---

### 3️⃣ שמירה ל-localStorage

```
בעת הוספת משתמש → נשמר ב-localStorage
בעת טעינת הדף → נטען מ-localStorage
בעת מחיקה → מתעדכן ב-localStorage
```

> ⚠️ **חשוב:** תטפל במקרה שבו אין עדיין נתונים ב-localStorage

---

### 4️⃣ רינדור נכון (בלי innerHTML +=)

**עזוב את** `innerHTML +=` ✋

**השתמש בחליפין:**
- `createElement`
- `appendChild`
- `DocumentFragment`

> **למה?** כי אתה צריך להתחיל לחשוב על:
> - ⚡ Performance
> - 🔄 מניעת reflow
> - 🎯 מניעת מחיקת event listeners

---

## 🧠 דגשים חשובים

אני רוצה לראות ממך:

- ✅ **פונקציה נפרדת** ל-`renderAllUsers()`
- ✅ **פונקציה נפרדת** ל-`renderSingleUser()`
- ✅ **מתודה** `deleteUser()` בתוך ה-`UserList` class
- ✅ **טעינה ראשונית** ב-`DOMContentLoaded`
